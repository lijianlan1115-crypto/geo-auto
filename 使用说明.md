# GEO反馈自动化：Chrome插件 + Python本地服务

这个目录分两部分：

- `python_service`：读取 Excel、保存截图、写回结果 Excel、记录进度。
- `chrome_extension`：打开 AI 平台网页、输入问题、等待回复稳定、最多追问 5 次、命中关键词后画红框并截图。

## Mac 开发测试步骤

1. 把 `input.xlsx` 放在 `chrome+python插件` 的上一级目录，也就是当前项目的 `geo数据/input.xlsx`。
2. 在 Mac 上运行：

   `python_service/启动服务_mac.command`

   如果双击打不开，可以在终端运行：

   ```bash
   cd "python_service"
   python3 -m venv .venv
   source .venv/bin/activate
   python -m pip install -r requirements.txt
   python server.py
   ```

3. 打开 Chrome，进入：

   `chrome://extensions/`

4. 打开右上角“开发者模式”。
5. 点击“加载已解压的扩展程序”，选择：

   `chrome_extension`

6. 先手动登录豆包、千问、DeepSeek、元宝、文心一言。
7. 在任意已匹配的 AI 平台页面左侧点击蓝色 `GEO` 小圆球，检查服务并开始测试。

## Windows 免 Python 使用步骤

适合交付给业务同事或普通 Windows 电脑使用。对方电脑不需要安装 Python。

1. 确认下面的程序位于 `python_service` 目录：

   `python_service/GEO反馈自动化.exe`

2. 双击运行：

   `python_service/GEO反馈自动化.exe`

3. 首次打开时，在界面中选择输入 Excel。表格可以放在电脑上的任意位置，不需要使用固定文件名或固定目录。

   程序会记住已选择的 Excel 路径并自动启动本地服务，不显示命令行黑窗。界面可查看服务状态、更换输入表格、重启服务、打开输入表格和输出目录，也可以设置登录 Windows 后自动启动。文件被移动或删除后，程序会提示重新选择。更换表格时，旧的结果表会自动归档到 `output/archive`。

4. 打开 Chrome，进入：

   `chrome://extensions/`

5. 打开右上角“开发者模式”。
6. 点击“加载已解压的扩展程序”，选择：

   `chrome_extension`

7. 先手动登录豆包、千问、DeepSeek、元宝、文心一言。
8. 点击 Chrome 右上角插件图标，先点“检查服务”，再点“开始运行”。

## Windows exe 打包步骤

这一步只需要打包人员做一次。普通使用者不需要执行。

1. 在 Windows 电脑安装 Python 3.10 或更高版本。
2. 双击运行：

   `python_service/build_windows_exe.bat`

3. 生成成功后会得到：

   `python_service/GEO反馈自动化.exe`

4. 把整个 `chrome+python插件` 文件夹交付给 Windows 用户。用户只需要双击：

   `python_service/GEO反馈自动化.exe`

注意：`pytesseract` 是截图 OCR 的备用检测方式。如果后续强依赖 OCR，需要在 Windows 电脑额外安装 Tesseract OCR；当前主流程优先用网页 DOM 文本检测和截图标注，不把 OCR 作为启动服务的硬性前置。

也可以在任意已匹配的 AI 平台页面左侧点击蓝色 `GEO` 小圆球：

- 配置 Python 服务地址
- 配置并发数
- 配置目标关键词
- 配置豆包、千问、DeepSeek、元宝、文心一言的平台 URL
- 检查服务、测试截图、开始、停止、刷新当前页、重载插件

插件后台打开任务标签页时，会优先使用小圆球里保存的平台 URL。

调试时：

- `使用本地模拟平台`：五个平台都打开本地 mock 页面，用于测试 Excel、截图、写回。
- `使用真实AI平台`：恢复豆包、千问、DeepSeek、元宝、文心一言真实 URL。

## 测试截图

Python 服务启动后，可以打开：

`http://127.0.0.1:8765/test-page`

页面左侧会出现 `GEO` 小圆球。点击后点“测试截图”。

目标关键词按以下顺序获取：优先使用插件面板填写的目标关键词；插件未填写时，再读取 Excel 当前行的 `关键词` / `目标关键词` / `keywords` / `keyword` 列。两处都未填写时，任务会停止并提示补充关键词。

测试截图会保存到：

`output/screenshots/test`

## 输出位置

- 结果 Excel：`output/result.xlsx`
- 截图目录：`output/screenshots`
- 进度数据库：`output/progress.sqlite`

中途断了可以重新启动服务和插件，已经完成的任务不会重复跑。

大批量任务支持断点续跑：每个平台、每个问题都会在 SQLite 中单独记录 `pending`、`running`、`done` 或 `failed` 状态。电脑重启、程序退出或 Chrome 中断后，重新打开程序并点击“开始 / 继续”，只会恢复中断和未完成的任务，已经完成的任务不会重复执行。不要点击“重置所有任务”，除非确实要从第一条重新开始。

SQLite 已嵌入 Windows EXE，客户电脑不需要安装 SQLite、数据库软件或 Python。需要换电脑继续时，先退出程序，再复制整个 `geo数据` 文件夹到新电脑；其中的 `output/progress.sqlite` 和 `output/result.xlsx` 就是断点状态与结果。新电脑选择同一份问题表后，程序会核对每一行问题，匹配成功就保留已有进度并继续执行；只有问题表内容不一致时才会归档旧结果并创建新任务。

## Mac / Windows 兼容说明

- Python 使用 `pathlib` 处理路径，不写死 Mac 或 Windows 路径。
- Chrome 插件不依赖系统路径，只通过 `http://127.0.0.1:8765` 访问本地服务。
- Mac 使用 `启动服务_mac.command` 或 `start_mac.sh`。
- Windows 普通用户双击 `GEO反馈自动化.exe`，图形界面会自动启动本地服务，不需要 Python 环境。
- Windows 开发模式下，如果没有 exe，`启动服务_windows.bat` 会自动在 `python_service/.venv` 创建虚拟环境。
- 输出目录、截图目录、进度数据库都在项目目录内，两个系统一致。

## 当前规则

- 目标关键词不设内置默认值，由插件面板或 Excel 每行提供。
- 插件面板里的目标关键词优先；插件未填写时，再读取 Excel 当前行的 `关键词` / `目标关键词` / `keywords` / `keyword` 列。
- 多个关键词是 OR 关系，可用 `，`、`,`、`、`、`;`、换行、`or` 分隔。
- 每个平台每个问题最多追问 5 次
- 如果命中关键词：滚动到关键词附近、画红框、截图、写回 Excel
- 如果 5 次追问后仍未命中：截最后页面，不标注，写回 Excel
- 等待回复完成方式：每 2 秒读取一次网页文本，连续 10 秒不变化认为回答完成，单轮最长等 180 秒

## 需要后续微调的地方

不同平台的网页 DOM 可能变化。如果某个平台提示“找不到输入框”，需要调整：

`chrome_extension/content_script.js`

里面的 `PLATFORM_RULES` 选择器。
